# Instalación

AgentFlow es una CLI en Go que une specs, investigación local, worktrees de Git, agentes de código externos y comandos de validación. La ruta de instalación es deliberada : obtiene un binario único, lo orienta al repositorio y la herramienta gestiona el ciclo de vida de ejecuciones, worktrees y estado en disco. Esta página cubre cómo construir o colocar el binario, inicializar `.agentflow/` en el proyecto y comprobar que Git, configuración y comandos agente están alineados antes de lanzar `work`.

## Requisitos

| Requisito | Notas |
| --- | --- |
| **Go 1.25+** | Build desde código o binario liberado |
| **Git** | Necesario para worktrees, estado y diffs |
| **Agentes configurados** | Como mínimo los comandos referenciados en `.agentflow/config.yaml` (`cursor-agent`, `kiro`, `ollama`, etc.) |

Opcional : **Ollama** para enriquecimiento o enrutado local y token **Notion** para sincronización (`NOTION_TOKEN`).

## Instalación desde fuentes

Clone el repositorio, compile con el `Makefile` del proyecto y use el binario bajo `bin/` :

```bash
git clone https://github.com/LaProgrammerie/hyper-fast-builder.git
cd hyper-fast-builder
make build
# binario: ./bin/agentflow
```

Añada `bin/` al `PATH` o copie `bin/agentflow` a cualquier directorio ya incluido en `PATH`.

## Inicializar un repositorio

Desde la raíz del proyecto ejecute `agentflow init`. Orquesta la superficie local de AgentFlow para que los comandos posteriores compartan un mismo fichero de configuración y SQLite.

```bash
agentflow init
```

Crea :

- `.agentflow/` — config, estado SQLite, runs, worktrees
- `.agentflow/config.yaml` — copia de `config.yaml.example` cuando falta
- `.agentflow/state.sqlite` — contabilidad de runs y tareas

Incorpore `.agentflow/state.sqlite` y `.agentflow/worktrees/` a `.gitignore`. `init` recuerda que esas rutas no deben versionarse.

## Verificar el entorno

Ejecute `agentflow doctor` tras instalar o si algo huele mal cableado. Doctor ejercita git, comprueba que la config existe, que los comandos agente resuelven en `PATH`, y muestra configuraciones típicamente rotas antes de malgastar tiempo en `work`.

```bash
agentflow doctor
```

Corrija lo reportado antes de usar `work`.

## Configuración

Copie y edite el ejemplo cuando necesite agentes, presupuestos o enrutado propios :

```bash
cp .agentflow/config.yaml.example .agentflow/config.yaml
```

Para cada bloque YAML y valor por defecto consulte [Archivo de configuración](/docs/es/configuration/config-file).

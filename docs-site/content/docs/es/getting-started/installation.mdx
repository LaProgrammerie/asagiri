# Instalación

Asagiri es una CLI en Go que une specs, investigación local, worktrees de Git, agentes de código externos y comandos de validación. La ruta de instalación es deliberada : obtiene un binario único, lo orienta al repositorio y la herramienta gestiona el ciclo de vida de ejecuciones, worktrees y estado en disco. Esta página cubre cómo construir o colocar el binario, inicializar `.asagiri/` en el proyecto y comprobar que Git, configuración y comandos agente están alineados antes de lanzar `work`.

## Requisitos

| Requisito | Notas |
| --- | --- |
| **Go 1.25+** | Build desde código o binario liberado |
| **Git** | Necesario para worktrees, estado y diffs |
| **Agentes configurados** | Como mínimo los comandos referenciados en `.asagiri/config.yaml` (`cursor-agent`, `kiro`, `ollama`, etc.) |

Opcional : **Ollama** para enriquecimiento o enrutado local y token **Notion** para sincronización (`NOTION_TOKEN`).

## Homebrew (macOS / Linux)

```bash
brew tap LaProgrammerie/tap
brew install asa
asa version
```

La fórmula se publica en [LaProgrammerie/homebrew-tap](https://github.com/LaProgrammerie/homebrew-tap) con cada tag de release.

## Instalación manual (binario publicado)

Descargue archivos y sumas desde [GitHub Releases](https://github.com/LaProgrammerie/asagiri/releases). Nombres tipo GoReleaser : `asa_Darwin_arm64.tar.gz`, `asa_Linux_x86_64.tar.gz`, `asa_Windows_arm64.zip`, etc.

```bash
VERSION=v0.1.0
curl -LO "https://github.com/LaProgrammerie/asagiri/releases/download/${VERSION}/checksums.txt"
curl -LO "https://github.com/LaProgrammerie/asagiri/releases/download/${VERSION}/asa_Darwin_arm64.tar.gz"
sha256sum -c checksums.txt
tar -xzf asa_Darwin_arm64.tar.gz
sudo install -m 755 asa /usr/local/bin/asa
asa version
```

Ajuste `VERSION`, el nombre del archivo y la ruta de instalación a su SO y CPU.

## Instalación desde fuentes

Clone el repositorio, compile con el `Makefile` del proyecto y use el binario bajo `bin/` :

```bash
git clone https://github.com/LaProgrammerie/asagiri.git
cd asagiri
make build
./bin/asa version
```

Añada `bin/` al `PATH` o copie `bin/asa` a cualquier directorio ya incluido en `PATH`.

## Inicializar un repositorio

Desde la raíz del proyecto ejecute `asa init`. Orquesta la superficie local de Asagiri para que los comandos posteriores compartan un mismo fichero de configuración y SQLite.

```bash
asa init
```

Crea :

- `.asagiri/` — config, estado SQLite, runs, worktrees
- `.asagiri/config.yaml` — copia de `config.yaml.example` cuando falta
- `.asagiri/state.sqlite` — contabilidad de runs y tareas

Incorpore `.asagiri/state.sqlite` y `.asagiri/worktrees/` a `.gitignore`. `init` recuerda que esas rutas no deben versionarse.

## Verificar el entorno

Tras cualquier vía de instalación (Homebrew, binario de release o `make build`), ejecute `asa doctor`. Doctor ejercita git, comprueba que la config existe, que los comandos agente resuelven en `PATH`, y muestra configuraciones típicamente rotas antes de malgastar tiempo en `work`.

```bash
asa doctor
```

Corrija lo reportado antes de usar `work`.

## Configuración

Copie y edite el ejemplo cuando necesite agentes, presupuestos o enrutado propios :

```bash
cp .asagiri/config.yaml.example .asagiri/config.yaml
```

Para cada bloque YAML y valor por defecto consulte [Archivo de configuración](/docs/es/configuration/config-file).

# Flujo consciente del coste

Guía operativa (`spec-doc` §11.4) con banderas y claves de configuración reales.

## 1. Estimación por tarea

```bash
agentflow estimate billing-v2 --task task-003 --dry-run
```

Muestra tokens de entrada/salida heurísticos y coste estimado según su hoja `pricing`.

## 2. Optimización de contexto

```bash
agentflow context billing-v2 --task task-003 --optimize
```

Construye un paquete de contexto reducido antes de prompts grandes al agente. Desactive la reducción con `agentflow work ... --no-context-reduction`.

## 3. Ejecución con presupuesto

```bash
agentflow work "develop billing-v2" --budget 0.50 --estimate-only
agentflow work "develop billing-v2" --budget 0.50
```

Los presupuestos de configuración (`budgets.per_run.max_estimated_cost`) aplican cuando la bandera CLI `--budget` es cero. Las ejecuciones que superan el presupuesto bloquean salvo:

- aceptación del aviso de confirmación, o
- `--yes` tras `BudgetPendingConfirmError`, o
- `--allow-over-budget` cuando `budgets.policies.allow_override_with_flag` es true

## 4. Preferencia local

```bash
agentflow work "develop billing-v2" --prefer-local --no-cloud
```

`--allow-cloud` permite explícitamente el enrutamiento a la nube cuando las políticas lo restringirían.

## Explicabilidad

`work` y `estimate` imprimen resúmenes de paso (elección de ruta, bytes de contexto, comprobaciones de presupuesto). Vea [Estimación de tokens](/docs/es/cost-performance/token-estimation).

# Flujo sensible al coste

Aquí **`spec-doc` §11.4** tiene equivalente ejecutable cada flag existe en builds vigentes orden estimación compaction budget preferencia locale según rutas config.

## 1 Estimar tarea puntual

```bash
asa estimate billing-v2 --task task-003 --dry-run
```

Muestra heurísticas IO tokens + cost estimado usando su `pricing`.

## 2 Optimizar paquete contexto

```bash
asa context billing-v2 --task task-003 --optimize
```

Construye packs reducidos antes prompts enormes puede desactivar `work ... --no-context-reduction`.

## 3 Correr dentro presupuesto

```bash
asa work "develop billing-v2" --budget 0.50 --estimate-only
asa work "develop billing-v2" --budget 0.50
```

YAML `budgets.per_run.max_estimated_cost` actúa con `--budget` cero Sobrecoste bloqueando salvo consenso usuario (`--yes` tras pendiente Budget) ó `--allow-over-budget` permisible tras policy.

## 4 Preferir localmente

```bash
asa work "develop billing-v2" --prefer-local --no-cloud
```

Combine `--allow-cloud` si estrategía exige reaperturas cloud conscientes.

## Explicabilidad

`work` / `estimate` imprimen rutas contextual bytes checkpoints ver [Estimación tokens](/docs/es/cost-performance/token-estimation).

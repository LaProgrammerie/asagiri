# Fuentes Notion

La integración Notion está en `application/internal/source/notion`. El invariante importante: tras el sync, ejecución e indexación leen **archivos locales** en disco, no la API Notion en cada paso. Trata el sync como un paso de snapshot que trae páginas al árbol de specs del repo.

## Configuración

Los nombres de propiedad deben coincidir con el esquema de tu workspace; Asagiri no adivina layouts de Notion.

```yaml
sources:
  notion:
    enabled: true
    token_env: NOTION_TOKEN
    default_database_id: ""
    specs_database_id: "<database-id>"
    tasks_database_id: ""
    status_property: Status
    title_property: Name
    updated_time_property: Last edited time
    import_path: .asagiri/specs
```

El token API se lee de la variable de entorno que nombra `token_env` (por convención `NOTION_TOKEN`).

## Comandos

Exporta el token, apunta `sync` a una URL de página o ejecuta sync completo, luego consume specs importadas vía `inbox` como cualquier otra fuente.

```bash
export NOTION_TOKEN="..."
asa sync notion --page "https://notion.so/..."
asa sync all
asa inbox --source notion
```

## Seguridad

Notion permanece **apagado** hasta `enabled: true`. Para ensayo: `asa --dry-run sync notion --page ...` sin efectos no deseados. Si ya existe una spec local y stdin es TTY, hay avisos antes de sobrescribir trabajo.

<Callout type="experimental">
Los esquemas API de Notion difieren por workspace. Valida nombres de propiedad (`status_property`, etc.) antes de confiar en sync en producción.
</Callout>

<Callout type="security">
Guarda tokens en entorno o gestor de secretos — nunca en YAML commiteado.
</Callout>

# Resumen de agentes

Asagiri es una capa de orquestación: no incluye pesos integrados ni acceso propietario a modelos. Usted declara **CLIs externas** bajo `agents:`; el enrutamiento, los valores por defecto de `work` o los indicadores por comando deciden qué binario se ejecuta en cada paso.

## Patrones habituales

La siguiente tabla enumera identificadores frecuentes; el ejecutable y los indicadores son siempre **los de su** `config.yaml`. El contrato de Asagiri consiste en lanzar `command` con `args` opcionales y reenviar flujos por la ruta habitual de redacción.

| ID de agente (ejemplo) | Rol típico |
| --- | --- |
| `kiro` | Redacción de especificaciones (`asa spec`) |
| `cursor` | Implementación (`dev`, agente por defecto de `work`) |
| `codex` / `claude` | Revisión o razonamiento exigente |
| `ollama` | Enriquecimiento local, clasificación, pre-revisión económica |

## Contratos

- Sin interpolación por *shell*: solo `argv`
- Tiempos de espera desde `agents.<id>.timeout`
- El *dry-run* omite la ejecución real
- Los registros pasan por redacción (`application/internal/redact`)

## Elegir agentes por paso

Cambie el ID de agente por subcomando cuando una tarea deba usar otra cadena de herramientas distinta de la predeterminada.

```bash
asa spec feat --agent kiro
asa enrich feat --agent ollama
asa dev feat --agent cursor
asa review feat --agent codex
```

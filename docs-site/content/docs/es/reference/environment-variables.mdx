# Variables de entorno

AgentFlow lee un conjunto corto explícito; sirve para que CI, shells y wrappers activen comportamiento sin reescribir configs.

| Variable | Efecto |
| --- | --- |
| `AGENTFLOW_DRY_RUN` | Con `1`, equivale a `--dry-run` global (capa exec puede fijarlo) |
| `AGENTFLOW_YES` | Con `1`, confirma automáticamente prompts (`requireConfirm`) |
| `NOTION_TOKEN` | Token por defecto si `sources.notion.token_env` es `NOTION_TOKEN` |
| `NOTION_TEST_PAGE_ID` | Solo pruebas de integración (`source/notion/integration_test.go`) |
| `UPDATE_GOLDEN` | Con `1`, refresca archivos golden del paquete `cost` |

## Token Notion renombrado

Si cambia nombre de secreto mantenga sincronizado `token_env` para que sync + API coincidan en la fuente.

```yaml
sources:
  notion:
    token_env: MY_NOTION_SECRET
```

```bash
export MY_NOTION_SECRET="..."
```

## Uso CI

`AGENTFLOW_DRY_RUN=1` habitual para ensayos orquestación sin binarios verdaderamente invocados. `AGENTFLOW_YES=1` resérvelo a jobs donde confirmaciones no debieran frenar después de chequear conscientemente límites de presupuesto.

```yaml
env:
  AGENTFLOW_DRY_RUN: "1"
```

Evite `AGENTFLOW_YES=1` salvo esa necesidad.

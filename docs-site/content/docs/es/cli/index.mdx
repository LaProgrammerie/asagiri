# Resumen de la CLI

El binario **`asa`** (`application/cmd/asa`) es la única superficie de entrada para inicializar proyectos, trocear el pipeline spec-driven en verbos discretos, inspeccionar artefactos de coste y tareas operativas. El tráfico diario suele ir por **`work`**, donde intenciones en lenguaje natural mapean pipelines enteros, o por pasos nombrados (**`plan`**, **`dev`**, **`verify`**, etc.) cuando `.kiro/specs/<feature>/` ya ancla tu plan. Los agrupamientos resumen la intención; los flags autoritativos vienen de Cobra vía la **[referencia CLI generada](/docs/es/cli/generated)** — regenera ese árbol cuando cambie el cableado CLI.

## Comportamiento global

Los flags de simulación aplican de forma coherente a subcomandos anidados:

| Flag | Descripción |
| --- | --- |
| `--dry-run` | Simula sin agentes externos ni shells de validación |
| (persistente) | Heredado por todos los subcomandos |

Variables de entorno como `ASA_DRY_RUN=1` y `ASA_YES=1` reflejan flags habituales; ver [Variables de entorno](/docs/es/reference/environment-variables) para la lista completa.

## Grupos de comandos de un vistazo

| Grupo | Comandos | Uso típico |
| --- | --- | --- |
| **Bootstrap** | `init`, `doctor`, `version` | Configuración inicial y comprobaciones de salud |
| **Pipeline** | `spec`, `plan`, `enrich`, `dev`, `verify`, `review`, `pr` | Pasos spec-driven en `.kiro/specs/<feature>/` |
| **Intent** | `work`, `continue`, `next`, `inbox` | Flujos en lenguaje natural y reanudación |
| **Sources** | `sync`, `index` | Fuentes externas (p. ej. Notion) e indexación |
| **Coste / contexto** | `estimate`, `investigate`, `context`, `cost`, `cost-report`, `cost-models` | Estimación, investigación, packs de contexto |
| **Inspect** | `inspect` (diff, symbol, tests) | Investigación local dirigida |
| **Ops** | `status`, `resume`, `report`, `clean` | Ciclo de vida de runs y artefactos |
| **MCP** | `mcp`, `mcp-serve` | Integración MCP si está activada |
| **Meta** | `docs`, `help` | Generación de docs y ayuda integrada |

Trata el agrupamiento como ayuda de navegación — no como mandato sobre cuántos comandos van en scripts frente al shell interactivo.

## Referencia generada por comando

Las páginas por comando con flags están en **[CLI generated](/docs/es/cli/generated)**. Tras cambiar comandos Cobra, regenera:

```bash
go run ./application/cmd/asa docs generate-cli \
  --output docs-site/content/docs/en/cli/generated
```

Haz commit del MDX generado con el cambio CLI para mantener la doc pública al día.

## Primeros comandos a probar

Prueba con seguridad antes de añadir agentes:

```bash
asa init
asa doctor
asa work "my feature" --dry-run --plan-only
```

Después sigue el [Inicio rápido](/docs/es/getting-started/quickstart) para un dry-run seguro, luego el [Primer flujo](/docs/es/getting-started/first-workflow) cuando agentes y comandos de validación estén configurados.

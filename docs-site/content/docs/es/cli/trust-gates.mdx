# `asa trust gates`

Muestra el perfil `verification` activo desde `.asagiri/config.yaml` y si las puntuaciones y checks requeridos pasarían (`spec-my-B` §19).

## Uso

```bash
asa trust gates
```

## Configuración

Gates bajo `verification.gates.<perfil>`:

```yaml
verification:
  default_profile: production
  gates:
    production:
      min_confidence:
        architecture: 0.8
        implementation: 0.75
        security: 0.85
      required_checks:
        - contracts
        - flows
        - observability
        - security
```

Cada clave `min_confidence` corresponde a una [dimensión de confianza](/docs/es/concepts/trust-engine#seis-dimensiones-de-confianza). `required_checks` deben completarse antes de pasar el gate.

# Runtime persistente

Asagiri evoluciona de una CLI sin estado a una **runtime de ingeniería local-first** (`spec-my-A` §24).

## Arquitectura

```text
asa daemon run → SQLite → sessions · events · memory · hooks
```

## Principios

Opcional, inspeccionable, seguro (solo `asa …` en hooks), memoria acotada.

## Modos

```yaml
runtime:
  mode: guided  # guided | interactive | headless | ci | review | exploration
```

## Embeddings de memoria

Memoria con `runtime.memory.embedder`: `hash` (por defecto), `ollama`, `cloud` (solo con `cloud.enabled: true`). Tras cambiar: `asa memory reindex`.

Ver [Configuración](/docs/es/configuration/config-file#runtime).

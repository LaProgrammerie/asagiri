# Onboarding del proyecto

El **onboarding del proyecto** prepara un template forkado (Go, PHP/Castor, Node) para que `asa work` y Mission Control funcionen sin editar `config.yaml` a mano.

```
asa onboard  →  detectar stack  →  merge config  →  bootstrap docs  →  ready
```

## Qué hace

- Detecta señales de stack (`go.mod`, `castor.php`, `package.json`)
- Fusiona comandos de validación de forma idempotente (preserva config custom)
- Bootstrap de `docs/ai/`, esqueleto Kiro, protegido por `--force-docs`
- Produce un **score de readiness** vía `asa ready`

## Qué no hace

- Generar features de negocio automáticamente
- Instalar binarios agent (solo detección + instrucciones)

## Paridad CLI

Cada acción TUI de onboarding tiene un equivalente CLI documentado (ADR-027).

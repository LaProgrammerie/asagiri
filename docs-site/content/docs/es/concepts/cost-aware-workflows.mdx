# Flujos conscientes del coste

## Problema

Las ejecuciones de agentes pueden consumir tokens de forma impredecible. Los equipos necesitan **previsiones**, **presupuestos** y **enrutamiento** que prefiera rutas más baratas cuando la calidad lo permite.

## Enfoque de AgentFlow

1. **Recuento heurístico de tokens** — ratios carácter/tipo de contenido (`token_estimation` en la config)
2. **Hoja de precios** — entradas manuales `pricing.models` para modelos en la nube (sin precios de proveedor en código)
3. **Presupuestos** — techos por ejecución, tarea y día con umbrales de confirmación opcionales
4. **Enrutamiento** — `routing.strategies` asocia clases de paso a agentes local vs nube rápida vs nube pesada

```bash
agentflow estimate billing-v2 --task task-003 --dry-run
agentflow work "develop billing-v2" --estimate-only
agentflow work "develop billing-v2" --budget 0.50 --allow-over-budget
```

<Callout type="cost">
Las estimaciones son **aproximadas**. La facturación del proveedor usa tokenizers reales; AgentFlow aún no incluye contadores exactos por proveedor.
</Callout>

## Compromisos

| Mejora | No resuelve |
| --- | --- |
| Sorpresas detectadas antes de `dev` | Facturas al céntimo |
| Política vía presupuestos YAML | Descubrimiento automático de precios en la nube |
| Razones de enrutamiento explicables | Selección óptima de modelo sin su configuración |

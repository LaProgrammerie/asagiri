# Sources Notion

Implémentation : `application/internal/source/notion`. L'exécution utilise toujours **des fichiers locaux** après synchronisation.

## Configuration

```yaml
sources:
  notion:
    enabled: true
    token_env: NOTION_TOKEN
    default_database_id: ""
    specs_database_id: "<database-id>"
    tasks_database_id: ""
    status_property: Status
    title_property: Name
    updated_time_property: Last edited time
    import_path: .agentflow/specs
```

Le jeton est lu depuis la variable d'environnement nommée par `token_env` (défaut `NOTION_TOKEN`).

## Commandes

```bash
export NOTION_TOKEN="..."
agentflow sync notion --page "https://notion.so/..."
agentflow sync all
agentflow inbox --source notion
```

## Sécurité

- Désactivé par défaut (`enabled: false`)
- Dry-run : `agentflow --dry-run sync notion --page ...`
- Invites d'écrasement lorsqu'une spec locale existe et que stdin est un TTY

<Callout type="experimental">
Les schémas API Notion diffèrent selon l'espace de travail. Validez les noms de propriétés (`status_property`, etc.) avant de dépendre de la sync en production.
</Callout>

<Callout type="security">
Conservez les jetons dans l'environnement ou un gestionnaire de secrets — jamais dans le YAML commité.
</Callout>

# Budgets

Implémenté dans `application/internal/cost/budget.go` et appliqué dans le pipeline V3 avant exécution agent.

## Schéma

```yaml
budgets:
  default_currency: EUR
  per_run:
    max_estimated_cost: 1.00
    max_estimated_tokens: 500000
    require_confirmation_above_cost: 0.20
  per_task:
    max_estimated_cost: 0.30
    max_estimated_tokens: 150000
  daily:
    max_estimated_cost: 10.00
  policies:
    block_when_over_budget: true
    allow_override_with_flag: true
    override_flag: "--allow-over-budget"
```

## Interaction CLI

| Drapeau | Comportement |
| --- | --- |
| `--budget <EUR>` | Plafond pour cette invocation |
| `--estimate-only` | Arrêt après estimation ; aucune dépense |
| `--allow-over-budget` | Contourne le blocage si la politique l'autorise |
| `--yes` | Accepte la confirmation après `BudgetPendingConfirmError` |

## Limites

Les estimations utilisent des jetons heuristiques × votre grille `pricing`. La facturation réelle fournisseur peut différer.

<Callout type="cost">
Configurez `pricing.models` avec des tarifs à jour. Des prix à zéro rendent l'affichage des coûts trompeur mais n'désactivent pas les plafonds de jetons.
</Callout>

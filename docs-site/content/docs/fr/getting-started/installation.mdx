# Installation

Asagiri est une CLI Go qui relie specs, investigation locale, worktrees Git, agents externes et validation. Un binaire unique, un dépôt cible, un état local `.asagiri/`.

## Ce qu’Asagiri est / n’est pas

- **Pas** un agent de chat de plus — un **control plane local-first**.
- **Orchestre** Cursor, Claude Code, Codex, Kiro, Ollama, etc.
- **Conserve** audit, ledger, replay et policies **sur votre machine**.

## Aucun compte requis

Pas de compte La Programmerie, pas de quota, pas de cloud obligatoire, pas d’envoi automatique des runs.

## Prérequis

| Élément | Notes |
| --- | --- |
| **Go 1.25+** | Uniquement pour compiler depuis les sources |
| **Git** | Worktrees, statuts, diffs |
| **Agents configurés** | Binaires sur `PATH` référencés dans `config.yaml` |

## Homebrew (macOS / Linux) — recommandé

```bash
brew tap LaProgrammerie/tap
brew install asagiri
asagiri version
asagiri doctor
```

Formule `asagiri` sur [LaProgrammerie/homebrew-tap](https://github.com/LaProgrammerie/homebrew-tap). Sur macOS, éviter `asa` si `which asa` → `/usr/bin/asa` ; alias optionnel : `alias asa='asagiri'`.

## Script d’installation

```bash
curl -fsSL https://raw.githubusercontent.com/LaProgrammerie/asagiri/main/scripts/install.sh | bash
```

Version épinglée : `ASAGIRI_VERSION=v1.0.0` (tag [GitHub Releases](https://github.com/LaProgrammerie/asagiri/releases)).

## Installation manuelle (archive)

Modèle GoReleaser v2 : `asa_<semver>_<os>_<arch>.tar.gz` — tag `v1.0.0` → `asa_1.0.0_darwin_arm64.tar.gz`.

```bash
VERSION=v1.0.0
VER="${VERSION#v}"
OS="$(uname -s | tr '[:upper:]' '[:lower:]')"
case "$(uname -m)" in
  x86_64|amd64) ARCH=amd64 ;;
  arm64|aarch64) ARCH=arm64 ;;
  *) echo "arch non supportée"; exit 1 ;;
esac
ARCHIVE="asa_${VER}_${OS}_${ARCH}.tar.gz"
BASE="https://github.com/LaProgrammerie/asagiri/releases/download/${VERSION}"

curl -fsSLO "${BASE}/checksums.txt"
curl -fsSLO "${BASE}/${ARCHIVE}"
grep " ${ARCHIVE}\$" checksums.txt | sha256sum -c -
tar -xzf "${ARCHIVE}"
install -m 755 asa "${HOME}/.local/bin/asa"
export PATH="${HOME}/.local/bin:$PATH"
asa version
```

## Depuis les sources

```bash
git clone https://github.com/LaProgrammerie/asagiri.git
cd asagiri
make build
./bin/asa version
```

## Initialiser un dépôt

```bash
asa init
asa doctor
asa agents list
asa agents sync --write
```

Alternative guidée : `asa onboard`. Ajoutez `.asagiri/state.sqlite` et `.asagiri/worktrees/` au `.gitignore`.

## Premier essai sans cloud

```bash
export ASA_DRY_RUN=1
asa work "explorer ma feature" --plan-only --yes
```

## Dépannage rapide

| Problème | Action |
| --- | --- |
| `command not found` | Vérifier le `PATH` |
| Agent manquant (`doctor`) | Installer le CLI ou éditer `config.yaml` |
| Specs agents absents | `asa agents sync --write` |
| Archive introuvable | Vérifier `asa_${VER}_${OS}_${ARCH}.tar.gz` pour le tag choisi |

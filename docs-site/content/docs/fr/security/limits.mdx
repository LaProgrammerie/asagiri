# Limites

Frontières de confiance explicites (`spec-doc` §17.1).

## AgentFlow ne

- **Garantit pas la correction** du code généré
- **Ne remplace pas votre suite de tests** — la validation exécute les commandes que vous configurez ; les échecs sont à vous corriger
- **Ne remplace pas la revue humaine** pour sécurité, conformité ou architecture
- **N'empêche pas toute injection de prompt** — specs malveillantes ou contenu Notion peut orienter les agents
- **Ne fournit pas une facturation jeton strictement alignée fournisseur**
- **N'offre pas de SLA production** — CLI OSS d'orchestration

## Ce qu'il fait

- Applique une **machine d'état** aux étapes du pipeline
- **Isole** les tâches dans des git worktrees
- **Estime** coût/jetons avant les exécutions
- **Bloque** certaines violations de politique (git dirty, noms de fichiers secrets, plafonds budget)
- **Enregistre** les exécutions pour audit (`report`, SQLite)

## Capacités expérimentales

Documentées séparément — surface MCP complète, sync Notion, réglage auto de la compression de contexte, reprise automatique hors dry-run, et RAG vectoriel.

<Callout type="warning">
Réussir `verify` signifie que les commandes configurées ont quitté avec le code 0 — pas que le changement est sûr à livrer.
</Callout>

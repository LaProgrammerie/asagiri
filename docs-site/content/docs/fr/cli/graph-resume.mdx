# `asa graph resume`

Continue un graphe en pause, en échec ou interrompu en utilisant les checkpoints persistés (`spec-my-C` §5.5, §15).

## Usage

```bash
asa graph resume graph-2026-05-29-9307e4be
asa graph resume graph-2026-05-29-9307e4be --json
```

## Comportement

Charge `execution-graph.yaml` depuis `.asagiri/graphs/<graph-id>/`, restaure les états des nœuds depuis les checkpoints et exécute les nœuds prêts restants. Émet des événements `graph.*` pendant la reprise.

Sans **checkpoint** persisté, la commande échoue avec un message explicite. Lancez d'abord [`asa graph run`](/docs/fr/cli/graph-run) avec `--checkpoint-every node` ou `group`.

Si `execution_graph.enabled: false`, les commandes graph (dont resume) renvoient une erreur.

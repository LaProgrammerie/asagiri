# `asa trust replay`

Rejoue une exécution antérieure depuis `.asagiri/trust/<trust-id>/replay.yaml` (`spec-my-B` §21). Utile pour auditer ce qui a été vérifié avant merge ou comparer après rebase.

## Usage

```bash
asa trust replay trust-2026-05-29-aefcf60d
asa trust replay trust-2026-05-29-aefcf60d --json
```

## Manifeste

`replay.yaml` contient :

- types de checks exécutés
- commit du dépôt
- contexte flow et branche
- commandes de validation associées

Après replay, de nouveaux `report.md` / `report.json` sont produits pour le nouvel identifiant d’exécution.

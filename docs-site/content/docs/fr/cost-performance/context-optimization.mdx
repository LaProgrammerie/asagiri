# Optimisation du contexte

Paquets : `application/internal/contextopt` — collecteur, score de pertinence, réducteur/compresseur, paqueteur.

## Pipeline

```mermaid
flowchart LR
  C[Collecte fichiers] --> R[Score de pertinence]
  R --> X[Compression / réduction]
  X --> P[Paquet de contexte]
  P --> A[Prompt agent]
```

## CLI

```bash
agentflow context billing-v2 --task task-003
agentflow context billing-v2 --task task-003 --optimize
agentflow work "develop billing-v2" --show-context-plan
```

`work` exécute l'optimisation de contexte dans le pipeline V3 sauf si `--no-context-reduction` est passé.

## Configuration

Les limites d'investigation (partagées avec grep local) sont sous `mcp.investigation` dans la config — elles s'appliquent même lorsque le serveur MCP est désactivé.

## Compromis

| Bénéfice | Limite |
| --- | --- |
| Prompts plus petits | Peut exclure des fichiers pertinents si les heuristiques manquent la cible |
| Appels cloud plus rapides | Ne remplace pas la lecture manuelle des chemins critiques |

<Callout type="experimental">
Les heuristiques de compression évoluent — comparez la sortie de `--show-context-plan` lorsque le contexte manque.
</Callout>

# Workflow CI

AgentFlow en CI doit rester **non interactif**, compatible **dry-run** et sans secrets dans les journaux.

## Modèle recommandé

```yaml
# extrait — adaptez à votre CI
- run: go build -o bin/agentflow ./application/cmd/agentflow
- run: bin/agentflow init
  working-directory: ${{ github.workspace }}
- run: bin/agentflow doctor
- run: bin/agentflow work "smoke feature" --dry-run --plan-only
- run: go test -race ./...
```

## Variables d'environnement

| Variable | Effet |
| --- | --- |
| `AGENTFLOW_DRY_RUN=1` | Équivalent au `--dry-run` global |
| `AGENTFLOW_YES=1` | Confirme automatiquement les invites (à utiliser avec prudence en CI) |
| `NOTION_TOKEN` | Requis uniquement pour les jobs de sync Notion |

Voir [Variables d'environnement](/docs/fr/reference/environment-variables).

## CI documentation

Ce dépôt construit le site public à chaque push :

```bash
cd docs-site && npm ci && npm run build
go run ./application/cmd/agentflow docs generate-cli --output docs-site/content/docs/en/cli/generated
go test ./application/internal/cli/docgen/...
```

Fichier workflow : `.github/workflows/docs.yml`.

## À éviter en CI

- Ne pas lancer de vrais `dev`/`review` sans agents épinglés et secrets
- Ne pas committer `.agentflow/config.yaml` avec des jetons
- Ne pas traiter `--estimate-only` comme un substitut aux tests pour `go test`

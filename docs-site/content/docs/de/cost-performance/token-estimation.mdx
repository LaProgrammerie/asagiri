# Token-Schätzung

Quellen: `application/internal/cost/token_counter.go` und `estimator.go`.

## Funktionsweise

1. Zeichenlängen für Dateien oder Textblobs messen
2. Inhaltstyp per Extension klassifizieren (`code`, `markdown`, `json`, `default`)
3. Durch `token_estimation.*_chars_per_token` teilen und aufrunden

```yaml
token_estimation:
  default_chars_per_token: 4.0
  code_chars_per_token: 3.2
  markdown_chars_per_token: 4.2
  json_chars_per_token: 3.6
```

## Kostenanzeige

Geschätzte Token × `pricing.models.<model-id>`-Tarife → geschätzte EUR (oder `budgets.default_currency`).

<Callout type="cost">
**Nicht anbieterexakt.** OpenAI, Anthropic und Google nutzen eigene Tokenizer. Zahlen als Größenordnungs-Schwellen behandeln.
</Callout>

## Befehle

```bash
agentflow estimate billing-v2 --task task-003
agentflow work "develop billing-v2" --estimate-only
agentflow cost billing-v2
```

## Golden Tests

Estimator-Form wird durch Golden Tests in `application/internal/cost/` abgedeckt. Goldens nur mit `UPDATE_GOLDEN=1` aktualisieren, wenn Algorithmen bewusst geändert werden.

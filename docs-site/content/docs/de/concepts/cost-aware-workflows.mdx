# Kostenbewusste Workflows

## Problem

Agent-Läufe können Token unvorhersehbar verbrauchen. Teams brauchen **Prognosen**, **Budgets** und **Routing**, das bei ausreichender Qualität günstigere Pfade bevorzugt.

## AgentFlow-Ansatz

1. **Heuristische Token-Zählung** — Zeichen-pro-Token-Verhältnisse nach Inhaltstyp (`token_estimation` in der Config)
2. **Preisliste** — manuelle `pricing.models`-Einträge für Cloud-Modelle (keine hart codierten Anbieterpreise im Code)
3. **Budgets** — Caps pro Lauf, Aufgabe, Tag mit optionalen Bestätigungsschwellen
4. **Routing** — `routing.strategies` mappt Schrittklassen auf lokale vs. Cloud-fast vs. Cloud-heavy-Agents

```bash
agentflow estimate billing-v2 --task task-003 --dry-run
agentflow work "develop billing-v2" --estimate-only
agentflow work "develop billing-v2" --budget 0.50 --allow-over-budget
```

<Callout type="cost">
Schätzungen sind **näherungsweise**. Anbieter-Abrechnung nutzt echte Tokenizer; AgentFlow liefert noch keine anbieterexakten Zähler.
</Callout>

## Kompromisse

| Verbessert | Löst nicht |
| --- | --- |
| Überraschungen vor `dev` | centgenaue Rechnungen |
| Policy über YAML-Budgets | automatisches Cloud-Pricing |
| erklärbare Routing-Gründe | optimale Modellwahl ohne Ihre Config |

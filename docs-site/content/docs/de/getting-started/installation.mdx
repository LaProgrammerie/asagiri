# Installation

AgentFlow ist eine Go-CLI, die Spezifikationen, lokale Voruntersuchung, Git-Worktrees, externe Coding-Agenten und Validierungsbefehle orchestriert. Binär installieren, Repository initialisieren, dann Umgebung prüfen.

## Voraussetzungen

| Voraussetzung | Hinweise |
| --- | --- |
| **Go 1.25+** | Aus Quellen bauen oder Release-Binär verwenden |
| **Git** | Für Worktrees, Status und Diffs erforderlich |
| **Konfigurierte Agents** | Mindestens die in `.agentflow/config.yaml` referenzierten Befehle (`cursor-agent`, `kiro`, `ollama`, usw.) |

Optional: **Ollama** für lokales Enrichment/Routing, **Notion**-Token für Sync (`NOTION_TOKEN`).

## Installation aus Quellen

```bash
git clone https://github.com/LaProgrammerie/hyper-fast-builder.git
cd hyper-fast-builder
make build
# Binär: ./bin/agentflow
```

`bin/` zum `PATH` hinzufügen oder `bin/agentflow` in ein Verzeichnis im `PATH` kopieren.

## Repository initialisieren

Im Projektroot:

```bash
agentflow init
```

Erzeugt u. a.:

- `.agentflow/` — Konfiguration, SQLite-Status, Läufe, Worktrees
- `.agentflow/config.yaml` — kopiert von `config.yaml.example`, falls fehlend
- `.agentflow/state.sqlite` — Lauf- und Aufgabenbuchhaltung

Stellen Sie sicher, dass `.agentflow/state.sqlite` und `.agentflow/worktrees/` in `.gitignore` stehen (`init` erinnert daran).

## Einrichtung prüfen

```bash
agentflow doctor
```

`doctor` prüft Git, Konfigurationsdatei, Agent-Befehle im `PATH` und typische Fehlkonfigurationen. Beheben Sie Meldungen vor `work`.

## Konfiguration

Bei Bedarf Beispiel kopieren und anpassen:

```bash
cp .agentflow/config.yaml.example .agentflow/config.yaml
```

Siehe [Konfigurationsdatei](/docs/de/configuration/config-file) für alle YAML-Blöcke.

# Installation

Asagiri ist eine Go-CLI für Specs, lokale Analyse, Git-Worktrees, externe Coding-Agenten und Validierung — ein Binary, ein Repo, lokaler Zustand unter `.asagiri/`.

## Was Asagiri ist / nicht ist

- **Kein** weiterer Chat-Agent — eine **local-first Control Plane**.
- **Orchestriert** Cursor, Claude Code, Codex, Kiro, Ollama usw.
- **Hält** Audit, Ledger, Replay und Policies **lokal**.

## Kein Konto erforderlich

Kein La-Programmerie-Konto, kein Kontingent, keine Pflicht-Cloud, kein automatischer Upload Ihrer Runs.

## Voraussetzungen

| Anforderung | Hinweise |
| --- | --- |
| **Go 1.25+** | Nur für Build aus Quellen |
| **Git** | Worktrees, Status, Diffs |
| **Konfigurierte Agenten** | Binaries auf `PATH` laut `config.yaml` |

## Homebrew (macOS / Linux) — empfohlen

```bash
brew tap LaProgrammerie/tap
brew install asagiri
asagiri version
asagiri doctor
```

Formel `asagiri` in [LaProgrammerie/homebrew-tap](https://github.com/LaProgrammerie/homebrew-tap). Unter macOS `/usr/bin/asa` umgehen — optional `alias asa='asagiri'`.

## Installationsskript

```bash
curl -fsSL https://raw.githubusercontent.com/LaProgrammerie/asagiri/main/scripts/install.sh | bash
```

Version pinnen: `ASAGIRI_VERSION=v1.0.0` ([GitHub Releases](https://github.com/LaProgrammerie/asagiri/releases)).

## Manuelle Installation (Archiv)

GoReleaser v2: `asa_<semver>_<os>_<arch>.tar.gz` — Tag `v1.0.0` → `asa_1.0.0_darwin_arm64.tar.gz`.

```bash
VERSION=v1.0.0
VER="${VERSION#v}"
OS="$(uname -s | tr '[:upper:]' '[:lower:]')"
case "$(uname -m)" in
  x86_64|amd64) ARCH=amd64 ;;
  arm64|aarch64) ARCH=arm64 ;;
  *) echo "unsupported arch"; exit 1 ;;
esac
ARCHIVE="asa_${VER}_${OS}_${ARCH}.tar.gz"
BASE="https://github.com/LaProgrammerie/asagiri/releases/download/${VERSION}"

curl -fsSLO "${BASE}/checksums.txt"
curl -fsSLO "${BASE}/${ARCHIVE}"
grep " ${ARCHIVE}\$" checksums.txt | sha256sum -c -
tar -xzf "${ARCHIVE}"
install -m 755 asa "${HOME}/.local/bin/asa"
export PATH="${HOME}/.local/bin:$PATH"
asa version
```

## Aus Quellen bauen

```bash
git clone https://github.com/LaProgrammerie/asagiri.git
cd asagiri
make build
./bin/asa version
```

## Repository initialisieren

```bash
asa init
asa doctor
asa agents list
asa agents sync --write
```

Alternativ: `asa onboard`. `.asagiri/state.sqlite` und `.asagiri/worktrees/` in `.gitignore`.

## Erster Lauf ohne Cloud

```bash
export ASA_DRY_RUN=1
asa work "explore my feature" --plan-only --yes
```

## Fehlerbehebung

| Problem | Lösung |
| --- | --- |
| `command not found` | `PATH` prüfen |
| Agent fehlt (`doctor`) | CLI installieren oder `config.yaml` anpassen |
| Agent-Specs fehlen | `asa agents sync --write` |
| Archiv nicht gefunden | `asa_${VER}_${OS}_${ARCH}.tar.gz` zum Release-Tag prüfen |

# Kostenbewusster Workflow

Betriebsleitfaden (`spec-doc` §11.4) mit echten Flags und Config-Keys.

## 1. Schätzung pro Aufgabe

```bash
agentflow estimate billing-v2 --task task-003 --dry-run
```

Zeigt heuristische Input-/Output-Tokens und geschätzte Kosten aus Ihrem `pricing`-Sheet.

## 2. Kontext-Optimierung

```bash
agentflow context billing-v2 --task task-003 --optimize
```

Erzeugt ein reduziertes Kontext-Pack vor großen Agent-Prompts. Reduktion deaktivieren mit `agentflow work ... --no-context-reduction`.

## 3. Lauf mit Budget

```bash
agentflow work "develop billing-v2" --budget 0.50 --estimate-only
agentflow work "develop billing-v2" --budget 0.50
```

Config-Budgets (`budgets.per_run.max_estimated_cost`) greifen, wenn CLI `--budget` null ist. Läufe über Budget blockieren außer:

- Bestätigungsaufforderung akzeptiert, oder
- `--yes` nach `BudgetPendingConfirmError`, oder
- `--allow-over-budget` wenn `budgets.policies.allow_override_with_flag` true ist

## 4. Lokale Präferenz

```bash
agentflow work "develop billing-v2" --prefer-local --no-cloud
```

`--allow-cloud` erlaubt explizit Cloud-Routing, wenn Policies sonst einschränken.

## Erklärbarkeit

`work` und `estimate` drucken Schritt-Zusammenfassungen (Routing-Wahl, Kontext-Bytes, Budget-Checks). Siehe [Token-Schätzung](/docs/de/cost-performance/token-estimation).

# Cost-aware workflows

## Problem

Agent runs can burn tokens unpredictably. Teams need **forecasts**, **budgets**, and **routing** that prefer cheaper paths when quality allows.

## AgentFlow approach

1. **Heuristic token counting** — chars-per-token ratios by content kind (`token_estimation` in config)
2. **Pricing sheet** — manual `pricing.models` entries for cloud models (no hardcoded vendor prices in code)
3. **Budgets** — per-run, per-task, daily caps with optional confirmation thresholds
4. **Routing** — `routing.strategies` map step classes to local vs cloud-fast vs cloud-heavy agents

```bash
agentflow estimate billing-v2 --task task-003 --dry-run
agentflow work "develop billing-v2" --estimate-only
agentflow work "develop billing-v2" --budget 0.50 --allow-over-budget
```

<Callout type="cost">
Estimates are **approximate**. Provider billing uses real tokenizers; AgentFlow does not yet ship provider-exact counters.
</Callout>

## Trade-offs

| Improves | Does not solve |
| --- | --- |
| Surprises caught before `dev` | Penny-perfect invoices |
| Policy via YAML budgets | Automatic cloud price discovery |
| Explainable routing reasons | Optimal model selection without your config |

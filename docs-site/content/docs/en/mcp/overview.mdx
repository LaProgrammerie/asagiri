# MCP overview

For editor bridges that speak **stdio JSON-RPC** MCP, Asagiri can expose a small server implemented in `application/internal/mcp`. It is intentionally narrow: list tools, call them with caps, and return structured results — not a full remote agent platform.

<Callout type="experimental">
MCP is disabled by default. Enable only in trusted environments after reviewing tool scopes.
</Callout>

## Enable

Toggle `mcp.enabled` and set safety knobs before you run anything that connects an external client to stdio.

```yaml
mcp:
  enabled: true
  max_output_bytes: 1048576
  command_timeout_seconds: 120
```

## Serve

`asa mcp-serve` runs the stdio transport; clients send newline-delimited JSON and expect JSON-RPC `initialize`, `tools/list`, and `tools/call`.

```bash
asa mcp-serve
```

Supports `initialize`, `tools/list`, `tools/call` (newline-delimited JSON).

## CLI wrapper

`asa mcp` is the helper that surfaces MCP-related subcommands for local inspection — pair it with the generated references when you need exact flags.

```bash
asa mcp
```

Inspect MCP-related commands — see generated [mcp](/docs/cli/generated/mcp) and [mcp-serve](/docs/cli/generated/mcp-serve).

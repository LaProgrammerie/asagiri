## When to use

Orchestration commands coordinate higher-order Asagiri workloads such as ingestion, syncing, backlog triage, investigative passes, hydrated context payloads, coarse estimates, and workspace-specific automation.

> Original CLI synopsis: Créer un replay package

## Usage

```bash
asa replay create [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--dry-run` | Inherited | `false` | Simuler les exécutions sans lancer d'agent ni commandes externes |
| `--from-graph` | Local | `(empty)` | Execution graph source id |
| `--from-investigation` | Local | `(empty)` | Investigation source id |
| `--from-run` | Local | `(empty)` | Run source id |
| `--include-events` | Local | `false` | Capturer events.jsonl |
| `--include-prompts` | Local | `false` | Capturer les prompts agents |
| `--include-runtime` | Local | `false` | Capturer les événements runtime |
| `--json` | Local | `false` | Sortie JSON |

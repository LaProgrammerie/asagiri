CLI path: **`asa knowledge query`**

## When to use

Orchestration commands coordinate higher-order Asagiri workloads such as ingestion, syncing, backlog triage, investigative passes, hydrated context payloads, coarse estimates, and workspace-specific automation.

> Original CLI synopsis: Interroger le graphe de connaissance

## Usage

```bash
asa knowledge query [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--dry-run` | Inherited | `false` | Simuler les exécutions sans lancer d'agent ni commandes externes |
| `--edge-type` | Local | `(empty)` | Filtrer par type d'arête |
| `--from` | Local | `(empty)` | Filtrer les arêtes sortantes depuis ce nœud |
| `--json` | Local | `false` | Sortie JSON |
| `--limit` | Local | `0` | Limite de résultats |
| `--max-depth` | Local | `0` | Profondeur BFS maximale |
| `--node` | Local | `(empty)` | Filtrer par ID de nœud |
| `--start` | Local | `(empty)` | Démarrer une traversée BFS depuis ce nœud |
| `--type` | Local | `(empty)` | Filtrer par type de nœud |

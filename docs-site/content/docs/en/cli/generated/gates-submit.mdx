> Original CLI synopsis: Write a human review verdict file for a task

## Usage

```bash
asa gates submit [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--dry-run` | Inherited | `false` | Simuler les exécutions sans lancer d'agent ni commandes externes |
| `--file` | Local | `(empty)` | Path to a full verdict YAML file |
| `--note` | Local | `(empty)` | Optional note when using --verdict |
| `--task` | Local | `(empty)` | Task ID (required) |
| `--verdict` | Local | `(empty)` | Verdict: pass, warn, or fail |

## Examples

```bash
asa gates submit human_review --task task-1 --verdict pass
  asa gates submit human_review --task task-1 --file ./review.yaml
```

CLI path: **`asa onboard`**

## When to use

Orchestration commands coordinate higher-order Asagiri workloads such as ingestion, syncing, backlog triage, investigative passes, hydrated context payloads, coarse estimates, and workspace-specific automation.

> Original CLI synopsis: Préparer le dépôt (config, docs, validation)

## Usage

```bash
asa onboard [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--check-only` | Local | `false` | Readiness seulement, sans écriture |
| `--ci` | Local | `false` | Mode CI (plain) |
| `--dry-run` | Local | `false` | Afficher les changements sans écrire |
| `--force-docs` | Local | `false` | Écraser docs substantiels |
| `--json` | Local | `false` | Sortie JSON |
| `--non-interactive` | Local | `false` | Sans prompts (requiert --yes) |
| `--plain` | Local | `false` | Sortie texte plain |
| `--resume` | Local | `false` | Reprendre depuis .asagiri/onboarding/state.json |
| `--stack` | Local | `auto` | Stack: auto\|go\|php\|node |
| `--step` | Local | `(empty)` | Aller à une étape (project\|stack\|agents\|docs\|feature\|validate) |
| `--strict` | Local | `false` | Traiter les warns comme des échecs |
| `--ui` | Local | `false` | Ouvrir le wizard TUI |
| `--yes` | Local | `false` | Accepter les valeurs par défaut |

## Examples

```bash
asa onboard --yes --non-interactive
  asa onboard --dry-run
  asa onboard --ui
```

## What happens internally

- Hydrates contextual services resolved from **$PWD**, coordinating git-bound automation, ingestion pipelines, syncing, backlog triage, investigations, MCP hooks, contextual hydration, coarse estimates, and similar orchestrations for this subtree.

## Failure modes

- Cobra rejects unknown flags or arity mismatches before any repository side effects occur.
- Asagiri cannot resolve scaffolding when the command is invoked outside an initialised checkout.
- Downstream automation may fail whenever credentials, network endpoints, or cached prerequisites drift from expectations.

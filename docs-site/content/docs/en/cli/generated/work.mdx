CLI path: **`asa work`**

## When to use

Orchestration commands coordinate higher-order Asagiri workloads such as ingestion, syncing, backlog triage, investigative passes, hydrated context payloads, coarse estimates, and workspace-specific automation.

> Original CLI synopsis: Exécuter une intention en langage naturel

## Usage

```bash
asa work [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--agent` | Local | `(empty)` | Agent d'implémentation |
| `--allow-cloud` | Local | `false` | Autoriser cloud explicitement |
| `--allow-over-budget` | Local | `false` | Dépasser le budget |
| `--budget` | Local | `0` | Budget run EUR |
| `--dry-run` | Inherited | `false` | Simuler les exécutions sans lancer d'agent ni commandes externes |
| `--estimate-only` | Local | `false` | Estimation sans exécution |
| `--investigate-first` | Local | `false` | Lancer une investigation avant le plan |
| `--investigate-on-failure` | Local | `false` | Investigation locale si l'exécution échoue |
| `--investigation-depth` | Local | `standard` | Profondeur investigation: quick\|standard\|deep\|ci |
| `--max-duration` | Local | `0` | Durée max (minutes) |
| `--max-input-tokens` | Local | `0` | Plafond tokens entrée |
| `--max-output-tokens` | Local | `0` | Plafond tokens sortie |
| `--max-tasks` | Local | `0` | Nombre max de tâches par run |
| `--no-cloud` | Local | `false` | Interdire cloud |
| `--no-context-reduction` | Local | `false` | Désactiver réduction contexte |
| `--no-review` | Local | `false` | Désactiver la review |
| `--plan-only` | Local | `false` | Afficher le plan sans exécuter |
| `--prefer-local` | Local | `false` | Préférer étapes locales |
| `--reviewer` | Local | `(empty)` | Agent de review |
| `--show-context-plan` | Local | `false` | Afficher chemin context pack |
| `--source` | Local | `(empty)` | Source externe (notion\|local) |
| `--stop-after` | Local | `(empty)` | Arrêter après une étape (verify, dev, …) |
| `--strict-trust` | Local | `false` | Après implémentation, enchaîner verify trust + gates |
| `--trust-flow` | Local | `(empty)` | Flow produit pour --strict-trust (défaut: feature résolue) |
| `--yes` | Local | `false` | Mode auto sans confirmation |

## Examples

```bash
# Feature technique
  asa work "ajoute export CSV" --dry-run

  # Intention produit — étape 1 (préparation)
  asa work "Créer un CRM pour artisans" --dry-run
  asa work "Créer un CRM pour artisans" --yes

  # Intention produit — étape 2 (implémentation)
  asa work crm-artisans --yes
  asa next --feature crm-artisans
```

## What happens internally

- Hydrates contextual services resolved from **$PWD**, coordinating git-bound automation, ingestion pipelines, syncing, backlog triage, investigations, MCP hooks, contextual hydration, coarse estimates, and similar orchestrations for this subtree.

## Failure modes

- Cobra rejects unknown flags or arity mismatches before any repository side effects occur.
- Asagiri cannot resolve scaffolding when the command is invoked outside an initialised checkout.
- Downstream automation may fail whenever credentials, network endpoints, or cached prerequisites drift from expectations.

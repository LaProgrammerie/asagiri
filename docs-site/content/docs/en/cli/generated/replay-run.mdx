CLI path: **`asa replay run`**

## When to use

Orchestration commands coordinate higher-order Asagiri workloads such as ingestion, syncing, backlog triage, investigative passes, hydrated context payloads, coarse estimates, and workspace-specific automation.

> Original CLI synopsis: Rejouer un workflow capturé

## Usage

```bash
asa replay run [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--compare` | Local | `false` | Préparer une comparaison avec la session précédente |
| `--dry-run` | Local | `false` | Valider le package sans réexécution |
| `--json` | Local | `false` | Sortie JSON |
| `--offline` | Local | `false` | Mode offline (pas d'appels cloud) |
| `--simulation` | Local | `false` | Rejouer graph/events sans agents |
| `--strict` | Local | `false` | Échouer si divergence détectée |

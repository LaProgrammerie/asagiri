CLI path: **`asa docs`**

## When to use

Use **docs** commands when refreshing machine-generated reference material—for example, syncing this MDX corpus with shipping CLI behaviour ahead of docs-site releases.

**generate-cli** is safe to run repeatedly; outputs are regenerated wholesale under the **--output** directory.

> Original CLI synopsis: Documentation tooling for Asagiri artefacts

## Usage

```bash
asa docs [flags]
```

## Options

| Flag | Scope | Default | Description |
| --- | --- | --- | --- |
| `--dry-run` | Inherited | `false` | Simuler les exécutions sans lancer d'agent ni commandes externes |

## Examples

```bash
asa docs
```

> Wire richer samples through **cobra.Command.Example** when the happy path warrants copy/paste-friendly automation beyond this scaffold.

## What happens internally

- Holds documentation-adjacent automation that should remain orthogonal to primary workflow commands (does not mutate SQLite artefacts).

## Failure modes

- Cobra rejects unknown flags or arity mismatches before any repository side effects occur.
- Asagiri cannot resolve scaffolding when the command is invoked outside an initialised checkout.
- Filesystem ACLs can deny writes under **--output**, leaving partial artefacts if interrupted mid-write.

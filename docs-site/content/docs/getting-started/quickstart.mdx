# Quickstart

This path uses **`--dry-run`** so you can explore orchestration without invoking external agents or validation shells.

## 1. Build and init

```bash
make build
export PATH="$PWD/bin:$PATH"
cd your-repo
agentflow init
agentflow doctor
```

## 2. Dry-run a natural-language workflow

```bash
agentflow work "develop my-feature" --dry-run --plan-only
agentflow work "develop my-feature" --dry-run --estimate-only
```

`--plan-only` resolves intent and prints the execution plan. `--estimate-only` runs the V3 pipeline through cost/context estimation without agent execution.

## 3. Inspect pipeline commands

With an existing Kiro spec folder (e.g. `.kiro/specs/my-feature/`):

```bash
agentflow plan my-feature --dry-run
agentflow enrich my-feature --dry-run
agentflow dev my-feature --dry-run
agentflow verify my-feature --dry-run
agentflow review my-feature --dry-run
agentflow status --dry-run
```

## 4. Reports

After a dry-run `work` creates run metadata:

```bash
agentflow status --dry-run
agentflow report <run-id> --dry-run
```

Reports land under `.agentflow/runs/<run-id>/report.md`.

## Run for real

Remove `--dry-run` only when agents and validation commands are configured and you accept external process execution:

```bash
agentflow work "develop my-feature" --estimate-only   # cost gate first
agentflow work "develop my-feature"                   # guided confirmation by default
```

<Callout type="warning">
Dry-run simulates orchestration; it does not prove agent output quality. Always run project validation (`go test`, linters, human review) on real changes.
</Callout>

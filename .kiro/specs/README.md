# Kiro specs

**Native** Kiro workflow artefacts for an initiative:

```
.kiro/specs/<feature-name>/
  requirements.md
  design.md
  tasks.md
```

## After every material change

1. Update `docs/ai/active/current-spec.md` (cross-tool summary).
2. Update `docs/ai/active/handoff.md` if execution changes.

See `docs/ai/context-map.md` for `.kiro/specs/` vs `docs/ai/active/` and anti-drift rules.
